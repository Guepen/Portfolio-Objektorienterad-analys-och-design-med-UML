Första modellen som gjordes

Så som vi har tänkt med denna modell är att på medlemsklassen så finns det ett attribut för roll för att skilja på de som är vanliga medlemmar och de som har lite mer ansvar i klubben som sekreterare.

En sekreterare kan boka händelser till exempel.

Alla medlemmar kan bara ha ett inlogg och de kan regristrera hur många båtar som helst. I båt-klassen har vi även efter disskusion med lärare lagt till metoder för att beskriva vad som en medlem(läs sekreterare) kan göra.

I bokningsklassen har vi gjort så att de metoder som finns där beskriver vilka händelser som kan bokas.

Till exempel boka händelse som bara en sekreterare kan göra och så vidare. Vid sidan av bokning så står det vad de olika metoderna kräver för att kunna fungera som till exempel boka en händelse så måste man vara sekreterare. Händelse klassen har vi kopplat till bokning då det finns ett attribut i bokning som heter book event och då tyckte vi att de skulle vara kopplade till varandra men det är bara sekreteraren som kan boka händelser.

Kalender klassen ansvarar bara för att visa händelser som sker i klubben.
